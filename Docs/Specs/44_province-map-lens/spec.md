# Spec: Province Map Lens

## Feature Intent

As a player exploring the map, I want a fourth "Province" map lens alongside Political, Geographic, and Org, so that I can see the sub-national province boundaries of the world while still understanding at a glance which country owns each province (via consistent country coloring).

## Acceptance Criteria

- **Given** the map lens switcher in the HUD bottom-left corner **When** the player opens it **Then** a fourth option, "Province", appears alongside Political, Geographic, and Org, with its own icon and localized tooltip label.
- **Given** the Province lens is selected for the first time in a session **When** province geometry has not yet been instantiated in the scene **Then** the game builds per-province render objects from `Assets/Configs/provinces_1880.json` (parallel to how `MapRenderer` builds per-country feature objects from `Assets/Map/world_1880.json`), without errors or a noticeable freeze that blocks input indefinitely.
- **Given** the Province lens is active **When** the map is rendered **Then** every province polygon is filled with the same color as its owning country would show under the Political lens (looked up via `CountryVisualConfig` using the province's `countryId`, from `province_config.json`) — so all provinces belonging to one country render in that single country's color, matching Political lens hues exactly.
- **Given** the Province lens is active **When** two provinces are adjacent (share a border), whether those provinces belong to the same country or different countries **Then** a gray outline line is visibly rendered along their shared edge.
- **Given** the Province lens is active **When** a province sits at the outer edge of a country/landmass (coastline, or border with an unrendered/ungenerated neighbor) **Then** a gray outline is still rendered along that edge, consistent with province borders elsewhere.
- **Given** the Province lens is active **When** a province's `generationMethod` is Micro (single-province country, e.g. Vatican, San Marino) **Then** it still renders with its country's Political color and a gray outline around its full country footprint, with no visual distinction from multi-province countries beyond having only one piece.
- **Given** the player switches away from the Province lens to Political, Geographic, or Org **When** the switch completes **Then** the map reverts to rendering country-level geometry exactly as it does today (province-level objects are not shown), and the existing three lenses' recoloring behavior is unchanged from current behavior.
- **Given** the player switches lenses repeatedly, including back to Province **When** switching **Then** performance stays interactive (no re-parsing of `provinces_1880.json` on every switch after the first activation — cached objects are reused).
- **Given** a country present in `country_config.json` has no corresponding entry in `province_config.json` (data gap) **When** the Province lens renders **Then** the missing country's territory does not crash the lens; it either falls back to showing the country-level geometry for that country only, or is left unfilled — this spec does not require new province data to be generated, only that the existing (already-complete, per `Docs/Specs/43_province-division/spec.md`) province dataset is rendered.
- **Given** per-province render objects (fill mesh + gray border mesh, built via a `MapMeshBuilder`-style triangulation of `provinces_1880.json`, analogous to how `MapRenderer` builds country features from `world_1880.json`) **When** the scene/map loads **Then** they are instantiated eagerly at startup alongside the existing country-level features, not lazily on first Province-lens activation, so switching to Province lens for the first time incurs no build hitch.
- **Given** province borders need a gray outline and the project has no existing `LineRenderer`/wireframe/outline infrastructure (confirmed: URP is fully active, custom shaders/Shader Graph are unused but available, and `MapMeshBuilder` only produces flat filled meshes today) **When** the border rendering is implemented **Then** it uses a procedurally-built border mesh per province ring — a quad-strip ("thick line") mesh generated from each ring's vertices at a fixed world-space line width, rendered with a simple unlit URP material in solid gray — mirroring the existing fill-mesh generation pattern in `MapMeshBuilder` rather than introducing `LineRenderer` components (avoids per-feature component overhead across potentially hundreds of provinces) or a Shader Graph outline effect (geometry-shader-style outline techniques are not reliably supported under URP on WebGL, the project's build target per `.claude/rules/unity/webgl.md`).
- **Given** the Province lens is active **When** the player hovers over or clicks a province **Then** the click/hover is tracked at province granularity (resolving `go.name` to the clicked province's `provinceId`) with a `// TODO` marking the point where province-selection behavior (info panel or other UI) will eventually hook in — no panel or visible selection UI is shown yet, but the interaction plumbing (click detection, provinceId resolution) is wired up so future work can attach behavior without re-deriving it.
- **Given** the Province lens is active **When** the player hovers over or clicks what would otherwise be a country-level feature (existing `MapClickHandler`/hover behavior) **Then** country-level interaction is suppressed for the duration the Province lens is active — country-level feature objects are hidden/disabled per the reversion criterion above, so their hover/click handlers naturally do not fire; no separate suppression flag is needed beyond hiding the country-level objects.

## Out of Scope

- A "selected province" info panel or any UI surface displaying province name/details. Click/hover tracking is wired up (see acceptance criteria) but terminates at a `// TODO` — no panel, tooltip, or highlight-on-select visual is implemented in this pass.
- Any new province metadata, geometry regeneration, or changes to the Province config pipeline (`scripts/generate_provinces.py`, `ProvinceProcessor`) — this feature only renders the already-generated `province_config.json` / `provinces_1880.json` output.
- Province-level gameplay systems: control, resources, characters, or actions tied to a province.
- Any change to the Org lens's per-country control-ratio coloring logic — Province lens coloring only mirrors Political lens hues, not Org tinting.
- Distinct border styling per adjacency type (e.g. thicker border between countries vs. thinner between same-country provinces) — a single uniform gray outline treatment for all province borders satisfies this spec.
- Deduplication of shared borders between adjacent provinces (i.e. each province ring may render its own border mesh independently, even where two provinces share an edge) — visual double-rendering of a shared border is acceptable and not treated as a defect for this pass.
