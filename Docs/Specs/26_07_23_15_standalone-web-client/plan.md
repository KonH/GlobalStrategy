# Plan: Standalone Web Client

## Spec

Source: `Docs/Specs/26_07_23_15_standalone-web-client/spec.md` (approved).

A standalone browser client runs the existing `src/` simulation without Unity. Screens: (A) main menu with NEW GAME / CONTINUE / SETTINGS — CONTINUE always loads the latest save, no picker; (B) organization selection via dropdown; (C) debug game view with MENU, PAUSE/PLAY, X1/X2/X3, the current date, the Unity-parity actions log, and an auto-completing terminal — no map; (D) in-game menu with SAVE and EXIT. Settings (language en/ru + auto-save interval) are their own persistent state, separate from saves, applied on every session start and deliberately overriding a loaded save's `AppSettings`. Every screen carries a localized header (experimental-version warning + link to the Unity WebGL demo) and footer (© 2026 KonH · MIT License + GitHub repo link). Terminal commands are reflection-discovered from `Game.Commands` so a new command ships with just a new build; Tab completes command names, then argument names, then labeled argument values driven by new `ParamSuggestionAttribute`-family annotations. Saves live per-browser; win/lose freezes the sim but saving still works. Deployment target is this repo's GitHub Pages project site (`https://konh.github.io/GlobalStrategy/`) via a GitHub Actions workflow.

## Goal

Ship `src/Game.WebClient/` — a Blazor WebAssembly standalone (.NET 8) debug client that reuses `GameLogic`/`BotSession`/`VisualState` unchanged, plus the additive `src/` pieces it needs (`StringConfig`, suggestion attributes), the `add-terminal-command` skill, and a GitHub Pages deploy workflow. All game rules stay in the existing ECS code; the web client is presentation and input glue only.

## Approach

Per the spec's Tech Notes (referenced, not restated):

- New Blazor WASM standalone project `src/Game.WebClient/` (net8.0, `Microsoft.NET.Sdk.BlazorWebAssembly`) referencing the existing `netstandard2.1` projects; IL interpreter mode, `RunAOTCompilation` off, `PublishTrimmed=false` so reflection (terminal discovery, `SaveSystem`'s `[Savable]` scan, `MakeGenericMethod` push) keeps working.
- Configs: the ten `Assets/Configs/*.json` files exposed as static web assets under `wwwroot/configs/`, fetched via `HttpClient`, deserialized by a new `StringConfig<TConfig>` beside `FileConfig` (shared `JsonSerializerOptions`); geojson through `GeoJsonParser.Parse`.
- Session: `GameLogicContext` built like `GameLifetimeScope` (player-first participating orgs), `GameLogic` + `BotSession.Create`, driven by a ~30 Hz timer loop passing wall-clock deltas; the loop keeps ticking while game-paused.
- Saves: `BrowserStorage : IPersistentStorage` — synchronous in-memory cache over an async IndexedDB write-behind (JS interop), preloaded at app start; web-side Newtonsoft `ISnapshotSerializer`. CONTINUE = `SaveFileManager.GetLastSave()` + `GameLogic.LoadState`.
- Settings: localStorage-backed preferences (language, auto-save interval), pushed as `ChangeLocaleCommand`/`ChangeAutoSaveIntervalCommand` on the first tick of every session.
- Localization: publish-time extraction of `Assets/Localization/{en,ru}.asset` into `wwwroot/locales/{locale}.json` by an MSBuild target in `Game.WebClient.csproj` invoking a small console tool (`src/Game.WebClient.LocaleTool/`); new `web.*` keys appended to both `.asset` files for header/footer.
- Terminal: reflection `CommandRegistry` + `key=value` parser + coercion + generic `Push<T>` executor in `Game.WebClient`; new `ParamSuggestionAttribute` family in `src/Game.Commands` retrofitted onto all existing commands; presentation-side suggestion providers keyed by attribute type; Tab completion per the spec's scenarios.
- Convention deliverable: `.claude/skills/add-terminal-command/SKILL.md` + `CLAUDE.md` index entry.
- Deploy: `.github/workflows/deploy-web-client.yml` publishing to the repo's GitHub Pages project site on push to `main`, `<base href>` rewritten to `/GlobalStrategy/`.

## Section 2 — User Steps

### 1. Enable GitHub Pages for this repository

In the GitHub repo settings (`Settings → Pages`), set **Build and deployment → Source** to **GitHub Actions**. Without this the deploy workflow cannot publish. The owner's existing user/organization Pages site is unaffected — this is the repo's own project site at `https://konh.github.io/GlobalStrategy/`.

### 2. Merge and watch the first deployment

After the implementation PR merges to `main`, open the Actions tab, confirm the `deploy-web-client` run completes green, and that the Pages deployment reports the project-site URL.

### 3. Smoke-test the deployed site in a browser

At `https://konh.github.io/GlobalStrategy/`: check header warning + full-client link and footer links on every screen; switch language and reload (choice persists); start a new game, use PAUSE/PLAY and X1/X2/X3, watch the actions log fill; run terminal Tab scenarios (command list, argument names, labeled country/org values); SAVE, EXIT, CONTINUE (state restored); leave it running unpaused past a month boundary and confirm CONTINUE picks up the autosave.

No Unity Editor steps are required: the only `Assets/` changes are plain-text locale-key appends and refreshed `Assets/Plugins/Core/` DLLs, both picked up automatically on the next Editor open.

## Tests

- `src/Game.Tests` (existing project):
  - `StringConfig` parity with `FileConfig`: for each real `Assets/Configs/*.json`, `StringConfig<T>(File.ReadAllText(...))` deserializes with results equivalent to `FileConfig<T>` (spot-check key fields per config type), proving both share one `JsonSerializerOptions`.
  - Suggestion attributes: shape tests (`AttributeUsage` targets, `OneOf` value retention) and the convention test — every public string field/positional property named `*Id` (plus `Locale`, `Interval`) on any non-abstract `ICommand` type carries a `ParamSuggestionAttribute` or `OneOf`. This mechanically enforces the retrofit and the skill's mandate for future commands.
- `src/Game.WebClient.Tests` (new project; can reference `Game.WebClient` because a Blazor WASM project is an ordinary net8.0 assembly):
  - `CommandRegistry`: discovers the full `ICommand` set (asserts representative names — `Pause`, `SaveGame`, `DebugImproveOpinion`), strips the `Command` suffix, models public fields and record positional properties with their types and attributes.
  - `TerminalParser`/`ValueCoercion`/`CommandExecutor`: valid `Name key=value` round trips into pushed commands (verified against a recording `IWriteOnlyCommandAccessor` fake); int/double/bool/enum coercion; unknown command, malformed argument, and missing required argument produce friendly errors and push nothing.
  - `SuggestionEngine` + providers: attribute type → provider mapping; labeled `Name (Id)` composition; enum auto-suggestion; unannotated parameter → empty suggestions; prefix filtering; providers exercised against the real `Assets/Configs/` JSONs with a stub locale. Presentation-side rendering of `Terminal.razor` itself is not unit-tested (no bUnit in scope) — covered by User Step 3.
  - `LocaleAssetParser`: parses the real `en.asset`/`ru.asset`, asserts known keys including quoted format values (`game_log.discovered_format`) and the new `web.*` keys; malformed lines are skipped without throwing.
  - `BrowserStorage` cache semantics against a fake `IStoragePersistence`: preload hydration, `Write`→`Read`/`Exists`/`List` synchronous visibility, `Delete`, `List` returning file names only, write-behind invocations recorded. The IndexedDB JS side is not unit-testable here — covered by User Step 3 (save → reload → CONTINUE).
  - `GameSession` headless: constructed from file-based sources, manual ticks advance `VisualState.Time`; first tick applies preferences (assert resulting `AppSettings` interval/locale override a loaded save's values); loop-while-paused lets `SaveGameCommand` process.
- Whole suite must stay green: `dotnet test src/GlobalStrategy.Core.sln` (existing `ECS.Tests`, `ECS.Viewer.Tests`, `Game.Tests` plus the new project), using the dotnet-test skill's log-file convention.
